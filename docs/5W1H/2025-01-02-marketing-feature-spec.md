# 이메일 시퀀스 엔진 업그레이드 - 마케팅팀 기능 명세서

> 작성일: 2025-01-02
> 대상: 마케팅팀, 영업팀
> 버전: 1.0

---

## 개요

### 무엇이 변경되나요?

이메일 캠페인(시퀀스) 발송 시스템이 **기존 60초 간격 처리 방식**에서 **실시간 이벤트 기반 처리 방식**으로 업그레이드됩니다.

### 한 줄 요약

> "이메일이 예약된 시간에 더 정확하게 발송되고, 캠페인 제어가 즉시 반영됩니다."

---

## 사용자 관점 변경사항

### 1. 이메일 발송 시간 정확도 향상

| 항목 | 기존 | 개선 후 |
|------|------|--------|
| **발송 지연** | 최대 60초 오차 | 1초 이내 정확도 |
| **예약 신뢰성** | 간헐적 지연 발생 가능 | 예약 시간 정확 보장 |

**예시**:
- 기존: 오후 2시 발송 예약 → 실제 발송 2:00:00 ~ 2:00:59 사이
- 개선: 오후 2시 발송 예약 → 실제 발송 2:00:00 정확히

### 2. 캠페인 제어 즉시 반영

| 동작 | 기존 | 개선 후 |
|------|------|--------|
| **캠페인 일시정지** | 최대 60초 후 반영 | 즉시 반영 |
| **캠페인 재개** | 최대 60초 후 발송 시작 | 즉시 발송 시작 |
| **리드 구독취소** | 최대 60초 후 반영 | 즉시 반영 |

**사용자 시나리오**:
```
[기존]
1. 긴급히 캠페인 일시정지 클릭
2. 최대 60초 동안 이메일 계속 발송됨
3. 의도치 않은 발송 발생 가능

[개선 후]
1. 캠페인 일시정지 클릭
2. 즉시 발송 중단
3. 정확한 제어 가능
```

### 3. 안전장치 강화

**새로운 안전 체크 항목**:

| 체크 항목 | 설명 | 결과 |
|----------|------|------|
| 캠페인 상태 | 캠페인이 'active' 상태인지 확인 | paused면 발송 스킵 |
| 리드 등록 상태 | 해당 리드의 등록이 'active'인지 확인 | stopped면 발송 스킵 |
| 리드 구독 상태 | 리드가 구독취소했는지 확인 | unsubscribed면 발송 스킵 |

**안전성 보장**:
- 일시정지된 캠페인의 이메일은 절대 발송되지 않음
- 구독취소한 리드에게는 절대 이메일이 발송되지 않음
- 법적 컴플라이언스 (GDPR, CAN-SPAM) 준수 강화

### 4. 발송 실패 시 자동 재시도

| 항목 | 기존 | 개선 후 |
|------|------|--------|
| **일시적 오류** | 실패 처리 | 자동 3회 재시도 |
| **재시도 간격** | 없음 | 30초 → 60초 → 120초 |

**예시**:
```
[기존]
이메일 서버 일시 오류 → 발송 실패 → 해당 리드 누락

[개선 후]
이메일 서버 일시 오류 → 30초 후 재시도 → 성공 → 정상 발송
```

## 기능별 상세 동작

### 캠페인 생성 및 리드 등록

**변경 없음** - 기존과 동일하게 동작

1. 캠페인 생성
2. 스텝(이메일) 추가
3. 리드 등록
4. 캠페인 시작

### 캠페인 일시정지 (Pause)

**동작 방식**:
1. 일시정지 버튼 클릭
2. 대기 중인 모든 이메일 발송 즉시 중단
3. 캠페인 상태: "일시정지됨"
4. 기존에 예정되어 있던 이메일들은 "보류" 상태로 유지

**참고**: 이미 발송 중인 이메일(현재 처리 중)은 완료됩니다.

### 캠페인 재개 (Resume)

**동작 방식**:
1. 재개 버튼 클릭
2. 보류 중이던 이메일들 즉시 활성화
3. 예약 시간이 지난 이메일: 즉시 발송 시작
4. 예약 시간이 남은 이메일: 해당 시간에 발송

### 캠페인 삭제 (Delete)

**동작 방식**:
1. 삭제 버튼 클릭
2. 대기 중인 모든 이메일 발송 취소
3. 캠페인 및 관련 데이터 삭제

### 리드 구독취소 (Unsubscribe)

**동작 방식**:
1. 리드 구독취소 처리
2. 해당 리드의 모든 대기 중인 이메일 즉시 취소
3. 모든 캠페인에서 해당 리드 발송 중단

## 영향 범위

### 현재 진행 중인 캠페인

**영향 없음** - 기존 데이터 100% 호환

- 진행 중인 캠페인 정상 유지
- 예약된 이메일 그대로 발송
- 추가 설정 불필요

### 배포 시 주의사항

**배포 시점에 발송 중인 이메일**:
- 안전하게 처리됨
- 중복 발송 없음
- 누락 없음

## FAQ

### Q1. 기존에 만들어둔 캠페인은 어떻게 되나요?
**A**: 그대로 유지됩니다. 수정이나 재설정 없이 정상 동작합니다.

### Q2. 배포 중에 이메일이 누락되거나 중복 발송되나요?
**A**: 아니요. 중복 방지 및 안전 체크 로직이 적용되어 있습니다.

### Q3. 일시정지했던 캠페인을 재개하면 어떻게 되나요?
**A**: 보류 중이던 이메일들이 즉시 활성화됩니다. 예약 시간이 이미 지났다면 바로 발송됩니다.

### Q4. 리드가 구독취소한 후에도 이메일이 발송될 수 있나요?
**A**: 아니요. 구독취소 즉시 모든 대기 중인 이메일이 취소되고, 시스템이 추가로 한 번 더 체크합니다.

### Q5. 이메일 발송 실패 시 어떻게 되나요?
**A**: 자동으로 3회까지 재시도합니다. 그래도 실패하면 "실패" 상태로 기록되어 확인 가능합니다.

### Q6. 발송 시간이 더 정확해지나요?
**A**: 네. 기존 최대 60초 오차에서 1초 이내로 개선됩니다.
