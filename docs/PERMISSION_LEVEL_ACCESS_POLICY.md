# Rinda 권한 레벨별 접근 정책

이 문서는 Rinda 서비스의 사용자 레벨(Level)별 접근 가능한 페이지, 메뉴, API 및 기능을 정의합니다.

---

## 1. 등급별 플랜 개요

| Level | 플랜 이름 | 가격 | 주요 대상 |
|-------|----------|------|----------|
| Level 1 | 체험판 유저 | 무료 | 기본 기능 체험 |
| Level 2 | 30만원 플랜 (Basic) | 30만원/월 | 소규모 비즈니스 / 대행 필요 고객 |
| Level 3 | 200만원 플랜 (Pro) | 200만원/월 | 직접 캠페인 운영 고객 |
| Level 4 | 엔터프라이즈 (Enterprise) | 협의 | 대규모/고급 기능 필요 고객 |
| Level 5 | 관리자 (Admin) | N/A | 시스템 및 워크스페이스 관리자 |

## 3. 레벨별 메뉴 접근 권한

### 접근 권한 매트릭스

| 메뉴 | Level 1 (체험판) | Level 2 (Basic) | Level 3 (Pro) | Level 4 (Enterprise) | Level 5 (Admin) |
|------|-----------------|-----------------|---------------|---------------------|-----------------|
| **대시보드** | ⚠️ 제한적 | ✅ | ✅ | ✅ | ✅ |
| **고객 탐색** | ❌ | ❌ | ✅ | ✅ | ✅ |
| **고객 관리** | ❌ | ❌ | ✅ | ✅ | ✅ |
| **캠페인** | ❌ | ❌ | ✅ | ✅ | ✅ |
| **답장 (인박스)** | ⚠️ 5회 제한 | ✅ 무제한 | ✅ | ✅ | ✅ |
| **Rinda GPT** | ❌ | ❌ | ❌ | ✅ 전용 | ✅ |
| **설정** | ⚠️ 제한적 | ✅ | ✅ | ✅ | ✅ |
| **워크스페이스 관리** | ❌ | ❌ | ❌ | ❌ | ✅ |
| **사용자 관리** | ❌ | ❌ | ❌ | ❌ | ✅ |

> ⚠️ = 제한적 접근 (일부 기능만 사용 가능)
> ✅ = 전체 접근
> ❌ = 접근 불가

## 4. 레벨별 상세 기능 권한

### Level 1 (체험판 유저)

**접근 가능:**
- 대시보드 (기본 통계만)
- 답장 인박스 (5회 열람까지)
- 설정 (개인 정보 수정만)

**제한 사항:**
- 5회 리드/답장 열람 시 **Paywall 팝업** 발생
- 고객 탐색/발굴 불가
- 캠페인 생성/관리 불가
- Rinda GPT 사용 불가

**특징:**
- 7일 체험 기간
- 결제 유도를 위한 제한적 데이터 노출

---

### Level 2 (30만원 플랜 - Basic)

**접근 가능:**
- 대시보드 (전체 통계)
- 답장 인박스 (무제한 열람)
- 설정 (전체)

**제한 사항:**
- 고객 탐색/발굴 불가 → **Level 5 관리자가 대행**
- 캠페인 직접 생성 불가 → **Level 5 관리자가 대행**
- Rinda GPT 사용 불가

**특징:**
- 리드/답장 무제한 열람
- 관리자 대행 서비스 포함

---

### Level 3 (200만원 플랜 - Pro)

**접근 가능:**
- 대시보드 (전체 통계)
- **고객 탐색** (AI 기반 리드 발굴)
- **고객 관리** (리드 CRUD)
- **캠페인** (시퀀스 생성/관리)
- 답장 인박스 (무제한)
- 설정 (전체)

**제한 사항:**
- Rinda GPT 사용 불가

**특징:**
- 셀프 서빙 가능
- 직접 고객 발굴 및 캠페인 운영

---

### Level 4 (엔터프라이즈)

**접근 가능:**
- Level 3의 모든 기능
- **Rinda GPT** (AI 채팅봇 전용)
  - 고객 탐색 지원
  - 고객 관리 지원
  - SQL 쿼리 생성 및 실행
  - 캠페인 자동 생성

**특징:**
- AI 기반 업무 자동화
- Rinda GPT 채팅 액션 제어

---

### Level 5 (관리자)

**접근 가능:**
- **모든 메뉴 및 기능**
- 워크스페이스 관리 (전체)
- 사용자 관리 (전체)
- Level 2 고객 대행 업무
- 시스템 설정

**특징:**
- 슈퍼 관리자 권한
- 모든 워크스페이스 데이터 접근

## 7. 구현 시 고려사항

### 프론트엔드

1. **라우터 가드**: 레벨별 접근 제어
   ```typescript
   // 현재 구현
   - ProtectedRoute: 로그인 필요
   - AdminProtectedRoute: admin 역할 필요
   - UserProtectedRoute: user 역할 필요

   // 추가 필요
   - LevelProtectedRoute: 특정 레벨 이상 필요
   ```

2. **사이드바 메뉴 필터링**: 레벨에 따른 메뉴 표시/숨김

3. **Paywall 컴포넌트**: Level 1 사용자 결제 유도

4. **기능별 버튼/액션 비활성화**: 권한 없는 기능 UI 처리

### 백엔드

1. **미들웨어 레벨 체크**: API 호출 시 사용자 레벨 검증

2. **데이터 필터링**: 레벨에 따른 데이터 접근 범위 제한

3. **사용량 추적**: Level 1 열람 횟수 카운팅

4. **워크스페이스 범위**: 본인 워크스페이스 데이터만 접근

## 8. 현재 DB 스키마의 userRole 매핑

| userRole | 대응 Level | 설명 |
|----------|------------|------|
| `user` | Level 1~4 | 일반 사용자 (플랜에 따라 레벨 결정) |
| `paying_user` | Level 2~4 | 유료 사용자 |
| `admin` | Level 5 | 워크스페이스 관리자 |
| `super_admin` | Level 5+ | 시스템 전체 관리자 |

> **Note**: 현재 `userRole`만으로는 Level 1~4를 구분하기 어려움.
> 별도의 `subscription_plan` 또는 `billing` 테이블과 연동 필요.

## 9. 향후 구현 계획

1. **구독/결제 테이블 연동**: 사용자의 실제 플랜 정보 관리
2. **레벨 기반 미들웨어**: API 호출 시 레벨 체크 자동화
3. **사용량 제한 시스템**: Level 1 열람 횟수 등 제한 구현
4. **Paywall 시스템**: 결제 유도 팝업 및 플로우
5. **권한 감사 로그**: 권한 관련 작업 추적
