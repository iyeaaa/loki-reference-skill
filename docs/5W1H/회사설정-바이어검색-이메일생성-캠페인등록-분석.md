## 1. 전체 아키텍처 개요

### 1.1 End-to-End 흐름

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            사용자 온보딩 플로우                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  [Step 1: 회사 설정]                                                         │
│       │                                                                     │
│       ├── 회사 기본정보 입력 (companyName, description, industry)            │
│       ├── 판매 전략 설정 (target, country, experience)                       │
│       └── AI 웹사이트 분석 (websiteAnalysis, targetAudiences)                │
│       │                                                                     │
│       ▼                                                                     │
│  [Step 2: 바이어 검색]                                                       │
│       │                                                                     │
│       ├── AI 쿼리 파싱 (industry, country 추출)                              │
│       ├── 하이브리드 검색 (BigQuery + Perplexity + Hunter.io)                │
│       ├── 이메일 Enrichment                                                  │
│       └── AI 점수 계산 및 리랭킹                                              │
│       │                                                                     │
│       ▼                                                                     │
│  [Step 3: 이메일 생성]                                                       │
│       │                                                                     │
│       ├── 템플릿 기반 생성 (변수 치환)                                        │
│       ├── AI 기반 생성 (GPT-4o)                                              │
│       └── 워크플로우 이메일 저장                                              │
│       │                                                                     │
│       ▼                                                                     │
│  [Step 4: 캠페인 등록]                                                       │
│       │                                                                     │
│       ├── 시퀀스 생성 (대상자 선택)                                           │
│       ├── 스텝 구성 (최대 4개)                                                │
│       └── 활성화 → Enrollment → 발송 스케줄링                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 기술 스택

| 계층 | 기술 |
|------|------|
| Frontend | React, React Query (TanStack Query), TypeScript |
| Backend | Elysia.js (Bun), TypeScript |
| Database | PostgreSQL + Drizzle ORM |
| AI/ML | OpenAI GPT-4o, Mastra Workflows, LangGraph |
| External APIs | BigQuery, Perplexity, Hunter.io, SendGrid |
| Queue | BullMQ (Redis) |

       │

CREATE TABLE emails (
  id UUID PRIMARY KEY,
  workspace_id UUID REFERENCES workspaces(id),
  user_email_account_id UUID REFERENCES user_email_accounts(id),
  lead_id UUID REFERENCES leads(id),

  body_text TEXT,

  -- draft, scheduled, queued, sent, delivered, opened, clicked, replied, bounced, failed

  -- 타이밍
