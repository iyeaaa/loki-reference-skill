## 5W1H 분석

### What (무엇을)

**현재 상태:**
- 체험판 온보딩 완료 시 발송되는 트랜잭셔널 이메일
- Loops.so API를 통해 발송
- 발견된 리드 수와 생성된 이메일 수를 표시
- 대시보드로 유도하는 CTA 버튼 포함

**문제점:**
| 항목 | 현재 | 업계 최적 | Gap |
|------|------|----------|-----|
| 제목줄 길이 | 7단어 | 2-4단어 | -7% 오픈율 |
| 개인화 수준 | 이름만 | 행동+산업 기반 | -31% 오픈율 |
| 긴급성 요소 | 없음 | 체험판 기한 표시 | -15% 클릭율 |
| 소셜 프루프 | 없음 | 고객 후기 1-2개 | -20% 전환율 |
| 진행률 표시 | 없음 | 단계별 표시 | -20% 완료율 |

### 모바일 제목줄 Truncation 문제

모바일 이메일 클라이언트는 제목줄을 **35-50자** 정도만 표시하고 나머지는 잘립니다.

**현재 제목줄 문제:**
```
데스크톱: [Rinda] 당신의 캠페인이 준비되었습니다!
          ────────────────────────────────────────
          전체 보임

모바일:   [Rinda] 당신의 캠페인이 준비되...
          ───────────────────────────────┘
          핵심 메시지가 잘림!
```

**문제점:**
- 사용자가 가장 먼저 보는 것이 `[Rinda]`라는 브랜드명
- 브랜드명은 이미 발신자 이름에서 보임 (중복)
- 정작 중요한 "캠페인이 준비되었습니다"는 잘려서 안 보임

**권장 해결책:**
```
개선 전: [Rinda] 당신의 캠페인이 준비되었습니다!
         ↓
개선 후: 철희님, 148명 바이어 발견

모바일:  철희님, 148명 바이어 발견
         ─────────────────────────
         핵심 메시지가 먼저 보임!
```

**핵심 원칙:**
가장 중요한 정보(개인화된 이름, 숫자)를 앞에 배치하면 모바일에서 잘려도 핵심이 전달됩니다.

**이메일 클라이언트별 제목줄 표시 길이:**
| 클라이언트 | 표시 길이 | 비고 |
|-----------|----------|------|
| iPhone (세로) | 35-40자 | 가장 짧음 |
| Gmail 앱 | 40-45자 | 중간 |
| Outlook 앱 | 45-50자 | 비교적 여유 |
| 데스크톱 | 60-70자+ | 충분함 |

### Why (왜)

**비즈니스 목표:**
- 체험판 → 유료 전환율 향상 (목표: 25%+)
- 장시간 온보딩 프로세스 중 사용자 이탈 방지
- 다음 단계(이메일 연결)로 유도

**개선이 필요한 이유:**

| 현재 지표 (추정) | 업계 벤치마크 | 최적 목표 |
|-----------------|--------------|----------|
| 오픈율 35% | SaaS 평균 29% | 50%+ |
| 클릭율 5% | SaaS 평균 4% | 12%+ |
| 전환율 15% | B2B 평균 14-25% | 25%+ |

**ROI 분석:**
```
현재: 1000명 체험판 × 15% 전환 = 150명 유료
개선: 1000명 체험판 × 25% 전환 = 250명 유료
증가: +100명 유료 고객 (+67%)
```

### Who (누가)

**발신자:**
- 시스템: Loops.so Transactional Email
- 발신명: "Rinda" (`support@rinda.ai`)

**수신자:**
- 체험판 온보딩을 완료한 사용자
- 특성: B2B 수출 기업, 영업팀 의사결정자
- 언어: 한국어(주) / 영어(부)

**관련 이해관계자:**
| 역할 | 책임 | 관심사 |
|------|------|--------|
| 마케팅팀 | 이메일 콘텐츠 | 전환율, 브랜드 일관성 |
| 개발팀 | 기술 구현 | 안정성, 유지보수성 |
| 제품팀 | 사용자 경험 | 완료율, 이탈률 |
| 경영진 | 비즈니스 성과 | 체험판→유료 전환율 |

### When (언제)

**현재 발송 시점:**
```
리드 발굴 완료 → 이메일 생성 완료 → 시퀀스 생성 완료 → 즉시 발송
```

**문제점:**
- 단일 이메일만 발송 (중간 이탈자 미대응)
- 최적 발송 시간 미고려

**업계 베스트 프랙티스:**
```
즉시:      "분석 시작" 알림
5분 후:    "진행 중" 업데이트 (중간 결과 미리보기)
완료 시:   "완료" 알림 (현재)
24시간 후: "아직 시작 안 하셨네요" (비활성 재참여)
3일 후:    "놓치기 아까운 리드들" (FOMO)
7일 후:    "체험판 만료 임박" (긴급성)
```

**최적 발송 시간:**
| 요일 | 시간 | 오픈율 |
|------|------|-------|
| 목요일 | 09-11시 | 44% |
| 화요일 | 09-11시 | 42% |
| 수요일 | 09-11시 | 41% |

### Where (어디서)

**발송 채널:**
- 현재: 이메일만 (Loops.so)
- 권장: 멀티채널 (이메일 + 인앱 알림 + 푸시)

**기술 스택:**
```

    ↓

    ↓

    ↓
사용자 인박스
```

**이메일 클라이언트 호환성:**
| 클라이언트 | 점유율 | 호환성 |
|-----------|--------|--------|
| Apple Mail | 46% | 양호 |
| Gmail | 28% | 양호 |
| Outlook | 12% | 양호 |
| 기타 | 14% | 테스트 필요 |

### How (어떻게)

**현재 구현:**
```typescript
// onboarding-worker.service.ts:1139-1161
if (isLoopsConfigured()) {
  await sendOnboardingCompleteEmail({

    firstName: user.username,
```

```
1. 헤더 (Rinda 로고, 그라데이션)

3. 인사말 (안녕하세요 {name}님)
4. 통계 카드 (리드 수 / 이메일 수)
5. CTA 버튼 (대시보드로 이동)
6. 다음 단계 (2개 항목)
7. 푸터 (문의처)
```
