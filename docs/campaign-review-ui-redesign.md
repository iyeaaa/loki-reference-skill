# Campaign Review UI/UX 재설계 문서

## 1. 개요

### 1.1 목적
캠페인 실행 전 사용자가 **리드 목록**과 **이메일 내용**을 모두 확인하고 수정할 수 있는 최적의 UI/UX 설계

### 1.2 현재 문제점

| 문제 | 현재 상태 | 영향 |
|------|----------|------|
| 리드 확인 불가 | Step 3에서 수신자 수만 표시 | 잘못된 대상에게 발송 위험 |
| 이메일 수정 불가 | Step 2로 돌아가야 수정 가능 | UX 흐름 단절, 비효율 |
| 미리보기 제한 | 본문 3줄만 표시 (`line-clamp-3`) | 최종 검토 불충분 |
| 정보 과부하 | 3열 레이아웃에 모든 정보 압축 | 가독성 저하 |

### 1.3 구조적 문제점 상세 분석

#### 프론트엔드 구조 문제

| 문제 영역 | 상세 내용 | 심각도 |
|----------|----------|--------|
| **컴포넌트 결합도** | Step 2의 `ManualModeContent.tsx`가 1400+ 라인으로 과도하게 큼 | HIGH |
| **상태 관리 복잡성** | `CreateCampaignPage`에서 모든 상태를 관리, Step 간 데이터 전달 복잡 | MEDIUM |
| **코드 중복** | Step 1과 Step 3에서 리드 표시 로직 중복 가능성 | LOW |
| **API 호출 비효율** | Step 3에서 리드 목록을 다시 조회하지 않고 수신자 수만 표시 | MEDIUM |

#### 데이터 플로우 문제

```
현재 플로우 (문제점):
CreateCampaignPage (모든 상태 관리)
    │
    ├── Step 1: customerGroupId, selectedLeadIds 설정
    │           → 리드 목록 조회 (useCustomerGroupMembersWithEmails)
    │           → 리드 선택/해제 가능
    │
    ├── Step 2: steps[] 이메일 스텝 작성
    │           → ManualModeContent (1400+ 라인 거대 컴포넌트)
    │           → 스텝 저장 시 API 호출
    │
    └── Step 3: 읽기 전용 요약만 표시 ⚠️ 문제
                → 리드 목록 없음 (수신자 수만 표시)
                → 이메일 수정 불가 (Step 2로 돌아가야 함)
                → 본문 3줄만 미리보기

개선된 플로우 (제안):
CreateCampaignPage
    │
    ├── Step 1: 고객 그룹 & 리드 선택 (현행 유지)
    │
    ├── Step 2: 이메일 스텝 작성 (현행 유지)
    │
    └── Step 3: 탭 기반 리뷰 UI (NEW)
                ├── Tab 1: 리드 목록 (검색/필터/선택)
                ├── Tab 2: 이메일 미리보기/편집
                └── Tab 3: 최종 확인 & 활성화
```

## 2. 현재 아키텍처 분석

### 2.1 파일 구조
```
admin/src/pages/sequences/
├── CreateCampaignPage.tsx        # 메인 페이지 (3단계 마법사)
├── CreateCampaignStep1.tsx       # Step 1: 고객 그룹 & 리드 선택
├── CreateCampaignStep2.tsx       # Step 2: 이메일 스텝 작성
├── CreateCampaignStep3.tsx       # Step 3: 검토 및 활성화 ⬅️ 개선 대상
└── components/
    └── ManualModeContent.tsx     # 이메일 편집기 컴포넌트
```

### 2.2 현재 Step 3 레이아웃
```
┌─────────────────┬─────────────────┬─────────────────┐
│  발송 대상 요약   │  이메일 시나리오  │   메모 & 액션    │

│   수신자 수)     │   3줄 미리보기)   │                 │
└─────────────────┴─────────────────┴─────────────────┘
```

### 2.3 데이터 플로우
```
CreateCampaignPage (campaignData state)
    ├── Step 1: customerGroupId, selectedLeadIds
    ├── Step 2: steps[] (이메일 스텝 배열)
    └── Step 3: 읽기 전용 요약 표시 → 저장/활성화
```

## 3. 제안 UI/UX 설계

### 3.1 디자인 원칙

1. **Progressive Disclosure**: 정보를 단계적으로 공개하여 인지 부하 감소
2. **Direct Manipulation**: 보이는 곳에서 바로 수정 가능
3. **Confirmation Before Action**: 중요한 액션 전 명확한 확인 단계
4. **Consistency**: 기존 컴포넌트 재사용으로 일관된 경험 제공

### 3.2 새로운 Step 3 레이아웃: 탭 기반 리뷰 UI

### 7.1 현재 사용 가능한 API 전체 목록

// Query Hooks (읽기)
useSequences(params?)              // 시퀀스 목록 조회

// Mutation Hooks (쓰기)
useCreateSequence()                // 시퀀스 생성
useUpdateSequence()                // 시퀀스 수정 ✅ Step 3에서 사용
useDeleteSequence()
useCreateSequenceStep(sequenceId?) // 스텝 생성 (파일 업로드 지원)
useUpdateSequenceStep(sequenceId?) // 스텝 수정 ✅ Step 3에서 사용 가능
useDeleteSequenceStep(sequenceId?)
useBulkEnroll()                    // 리드 일괄 등록

// Query Hooks
useCustomerGroups(params?)
